## 这版的浅 / 深主题可读性打磨

四处改动，全靠给 `<html>` 加 `data-mode=light|dark`（在 `themes-patch.js` 的 `applySkin` 里设），CSS 用 `[data-mode=...]` 圈作用域，只改该改的、不波及另一边：

1. **删终端活动标签底部的项目色横线**（`soft-patch.css` 的 `.term-tab.active::after`）。活动标签本就有「左缘色脊 + 整块背景填充」两个指示器，底部再加下划线是第三个，冗余。深浅都删。
2. **浅色终端标签底色改淡染**：原来填 `hsl(--th 50% 50%)` 的项目色块，在浅底上发闷像深补丁。浅色皮肤改成高亮度低不透明度的淡染，hover/active 逐档加深，左脊保留当重点。深色不动。
3. **浅色用量板块弱字提对比**：卡内局部把 `--text-dim` / `--text-faint` 重定义得更深（`#usage-sec` 上设变量，下面 `.ux-*` 自动继承）。
4. **深色用量板块弱字轻提一档**：同病根但更轻，只动 `--text-faint`。

### 弱字对比度（数值，9 套浅 + 9 套深皮肤全量校验过）

| 档位 | 浅色改前 | 浅色改后 | 深色改前 | 深色改后 |
|---|---|---|---|---|
| faint（重置倒计时 / 单位 / 小注） | ~2.3 | 4.0–6.2 | ~2.8 | ~4.2 |
| dim（窗口标签） | ~3.9 | 5.7–9.8 | ~5.3 | 不动 |
