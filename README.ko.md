# Rurutia

**Coding Agent를 위한 콕핏 —— [FanBox](https://github.com/alchaincyf/fanbox) 기반 개인 강화판**

Claude Code / Codex가 로컬에서 작업하도록 지휘하고, 그것이 건드린 모든 파일과 고친 모든 줄을 분명히 확인하며, 언제든 이어받으세요.<br>
그 위에 Rurutia는 비주얼과 폰트를 다시 만들고, **18종 컬러 스킨**, **Starship 내장 터미널 프롬프트**, **터미널 브랜드 아이콘 툴바**를 추가했으며, 사이드바 진입점·사용량 패널·인터랙션 디테일을 더 손에 맞게 다듬었습니다.

## 이게 무엇인가요

[**FanBox**](https://github.com/alchaincyf/fanbox)(제작: [Huashu](https://github.com/alchaincyf))는 로컬에서 실행되는 「**Coding Agent 콕핏**」입니다: 로컬 파일을 탐색 / 미리보기 / 편집하는 동시에, 내장된 진짜 터미널에서 Claude Code, Codex 또는 어떤 coding agent든 실행하고, agent가 어느 파일을 고치면 실시간으로 하이라이트됩니다 —— **파일 되찾기 → agent 실행 → 변경 확인**, 모두 한 창에서 완료됩니다. 무의존 백엔드, 데이터는 로컬을 벗어나지 않습니다.

> *"AI가 한나절 만에 프로젝트 열 개를 만들어 주고, 그다음엔 다시는 찾을 수 없게 됩니다. FanBox는 그걸 되찾아 줍니다."*

**Rurutia**는 제가 FanBox v2.3.1(`c93a486`)을 기반으로 만든 **개인화 개조판**입니다: 핵심 기능은 100% FanBox에서 왔고, 저는 비주얼 / 폰트 / 컬러를 다시 만들고, 스킨과 터미널 프롬프트 시스템 한 벌을 통째로 추가했으며, 일상적인 편의성을 수십 군데 다듬었습니다. 아래에서 먼저 **원본 프로젝트의 전체 기능**을 설명하고, 그다음 **제가 구체적으로 무엇을 바꿨는지** 이야기합니다.

## 30초 만에 이해하기

| 하고 싶은 것 | Rurutia에서는 |
|---|---|
| 한나절 만에 마구 만든 프로젝트 열 개 되찾기 | `⌘K` 전역 퍼지 검색 · 폴더에 node/web/py/rs/go 배지를 달아 타입을 한눈에 식별 |
| agent에게 일을 시키면서, 무엇을 고쳤는지도 분명히 확인하기 | 내장된 진짜 터미널에서 Claude Code / Codex 실행; 어느 파일을 쓰면 그 카드가 즉시 빛나고 미리보기가 실시간으로 따라옴 |
| 어제 세션 이어가기 | 프로젝트를 열어 지난 세션을 보고, 「▶ 이어가기」 한 번으로 `claude --resume` / `codex resume`로 컨텍스트 복귀 |
| 공식 사용량을 주시하며 한도 초과 막기 | 사이드바에 Claude / Codex의 5시간 윈도우 + 주간 할당량을 상시 표시, 한도에 근접하면 빨간 막대 + 데스크톱 알림 |
| 기분에 따라 전체 화면 갈아입히기 | 18종 컬러 스킨 + 16종 터미널 프롬프트 테마, UI / 터미널 / 코드 하이라이트가 함께 변경 |

## 원본 FanBox로 할 수 있는 것 (전체 기능)

> 이 부분은 FanBox 자체의 기능으로, Rurutia가 그대로 유지합니다. 원본 영문 설명은 [`README.fanbox.md`](README.fanbox.md)를 참고하세요.

### 🗂 파일 · 되찾기와 미리보기
- **⌘K 전역 퍼지 검색**: 이름 일부만 기억하면 됩니다; `⌘↵`로 에디터에서 프로젝트를 통째로 열기; `内容:` 접두어(예: `内容:키워드`)로 전문 검색 전환.
- **선명한 컬러 실물 아이콘**: 모든 파일이 「자기 자신처럼 보입니다」 —— PDF는 빨강, JS는 노랑, Markdown은 파랑; 사진과 영상은 실제 비율로 표시됩니다.
- **제자리 미리보기**: Markdown 렌더링, HTML 실시간 완성본, 코드 구문 강조, 이미지/영상/PDF 내장(HEIC 포함), 압축 파일 내용 목록, 투명 이미지 체커보드 배경.
- **썸네일 가속**: 큰 폴더에서도 스크롤과 클릭이 0.1초 이내.
- **프로젝트 배지**: 폴더 카드에 node / web / py / rs / go를 표시해, 한나절에 만든 프로젝트 열 개도 타입을 한눈에 식별.

### 👀 agent가 무엇을 고쳤는지 보기
- **살아있는 대시보드**: agent가 파일을 하나씩 쓸 때마다 그 카드가 즉시 물결을 일으키고, 변경 빈도에 따라 빛을 내며 호흡하듯 깜빡입니다. agent가 쓰는 곳을 따라 빛도 움직입니다.
- **팔로우 모드**: 한 번의 클릭으로 파일 뷰 + 미리보기가 agent가 편집 중인 파일을 추적합니다 —— 코드는 새로 쓰인 줄을 따라 하이라이트되며 깜빡이고, HTML은 작성하는 대로 실시간 렌더링(더블 버퍼링, 흰 화면 깜빡임 없음), Markdown도 실시간 렌더링됩니다. 직접 둘러보기 시작하면 즉시 제어권이 다시 당신에게 돌아옵니다.
- **세션 리플레이**: 영상을 넘기듯 타임라인을 드래그해, 그 시간 동안 agent가 어떤 파일을 단계별로 고쳤는지 재현합니다.
- **변경 사항 수신함**: 여러 프로젝트에 걸쳐 이번 세션에서 변경된 모든 파일을 모아 봅니다.
- **Git 변경 diff**: Monaco 읽기 전용 DiffEditor로 HEAD와 현재 작업 트리를 나란히 표시합니다.

### 🤖 Agent 콕핏
- **프로젝트 메모리**: 어떤 프로젝트 폴더든 열어 AI가 여기서 무엇을 했는지 확인하세요 —— 지난 세션(당신의 첫 문장이 제목), 매 세션에서 고친 파일, 트리거된 skill; 「▶ 이어가기」 한 번으로 내장 터미널에서 `claude --resume` / `codex resume`로 컨텍스트를 복귀합니다.
- **스크린샷 다이렉트**: 시스템 스크린샷이 저장되는 즉시 다이렉트 카드가 떠오릅니다 —— 터미널의 agent에 전달하거나, 프로젝트 `소재/`에 담거나, 먼저 주석을 단 뒤 보낼 수 있습니다.
- **AI 정리**: AI는 메타데이터만 보고 정리 제안을 내며(내용을 읽지 않고, 파일 시스템을 건드리지 않음), 한 건씩 사람이 검토한 뒤 실행하고 + 롤백 로그를 기록해 한 번에 전체 되돌리기가 가능합니다.
- **릴리스 마법사**: node 프로젝트에서 버전 번호, CHANGELOG, 패키징, 푸시, GitHub Release를 한 번에 엮어 줍니다.
- **Skills 살펴보기**: 로컬의 모든 agent skill을 한 화면에 —— 트리거 통계, 헬스 체크, context 예산, 파일을 지우지 않는 켜기/끄기 스위치.
- **Agent 사용량**: Claude Code 공식 5시간 윈도우/주간 할당량(`/usage`와 동일 출처) + 로컬 token 통계; Codex 한도 스냅샷.
- **디스크 사용량 살펴보기**: `du` 기준의 실제 사용량 막대 순위, 드릴다운 가능.

### 🖥 터미널 · agent 지휘
- **진짜 내장 터미널**: node-pty + xterm.js(WebGL 렌더링), Claude Code / vim / htop를 실행해도 화면이 깨지지 않고, 한중일 전각 문자도 정확합니다.
- **파일을 터미널로 드래그**: 파일 목록에서 파일/폴더를 터미널로 끌어다 놓으면 경로가 자동으로 삽입되어 agent에 컨텍스트로 전달됩니다.
- **클릭 가능한 경로**: 터미널에 나타난 파일 경로를 바로 열 수 있습니다(공백이 포함된 스크린샷 이름, 한자 이름, 줄바꿈된 긴 경로도 인식).
- **선택만 하면 터미널로 전송**: 미리보기에서 텍스트 일부를 선택하면, 「파일 출처 + 코드 펜스」 형식으로 한 번에 터미널로 보냅니다.
- **상황 인식**: 탭의 동그란 점이 agent의 실행/대기/종료 상태를 표시; 당신 차례가 되면 터미널 가장자리가 호흡하듯 알리고, 긴 작업이 끝나면 시스템 알림을 보냅니다.

### ✍️ 편집 · 보이는 그대로(WYSIWYG)
- **Markdown**: Milkdown Crepe가 Notion 스타일의 보이는 그대로 편집을 제공하고, 입력을 멈춘 지 0.8초 후 자동 저장됩니다.
- **코드/JSON**: Monaco 에디터(VS Code와 동일한 코어).
- **이미지 주석**: 펜/화살표/텍스트/모자이크, 포맷 변환, 압축, 해상도 조정.
- **미저장 가드**: 세 가지 에디터 모두 저장하지 않은 채 나가는 것을 일괄 차단합니다.

### 🎨 18종 컬러 스킨(다중 강조색 시스템)

사이드바의 「스킨」을 누르면 색상 카드 그리드가 떠 전환할 수 있으며, 기본값은 「픽셀 라이트」입니다. 각 스킨은 「**중성 바탕 + 나란한 강조색 3개**(메인: 버튼/활성 상태 · 서브: 섹션 제목과 링크 · 포인트: 배지) + 한 벌의 시맨틱 상태색」으로 구성됩니다 —— 색은 더 선명하면서도, 역할 분담으로 조화를 유지합니다. 본문 / 강조색 / 배지 글자 / 터미널 16 ANSI **모두 WCAG 대비 검증을 통과**합니다; 각 스킨은 메인 화면, 사이드바, 터미널 색상, 코드 하이라이트에 자동으로 맞춰지고, Monaco 에디터 배경색까지 같은 색온도를 따라갑니다.

영감은 WeChat 공식 계정 「色所」의 고급스러운 컬러 모음에서 받았습니다: 네오 멤피스 / 애시드 패션 / 디지털 젤리 / 회로 기판 / 보이드 / 라바 오렌지 / 심해 코어…… 라이트 9종 다크 9종, 총 18종.

### 🚀 터미널 프롬프트(Starship 내장 · 16종 테마)

**바로 쓸 수 있는 powerline 프롬프트**: 서명·공증된 starship + Nerd Font 아이콘 폰트가 내장되어, 설치 후 터미널을 열면 곧바로 powerline 알약 모양(디렉터리 / git 상태 / 언어 버전 / ♥ 시간)이 나옵니다. **직접 starship을 설치할 필요도, `~/.zshrc`를 설정할 필요도 없습니다**.

ZDOTDIR 주입 방식: 먼저 당신의 실제 dotfile을 source하고(PATH / 별칭이 한 치의 오차도 없이 유지되어 `claude` / `codex`도 그대로 찾을 수 있음), 그 위에 starship을 얹습니다 —— **이 App 터미널에서만 적용되고, 어떤 dotfile도 건드리지 않으며, 제거 시 잔여물이 전혀 없습니다**. macOS + zsh 전용.

<p align="center"><sub>▲ 사이드바 「프롬프트」 선택기: 전체 테마는 하나만 고르고, 겹쳐 적용하는 보정은 여러 개 선택 가능; 전환은 즉시 적용되며, 실행 중인 터미널은 엔터를 누르면 모습이 바뀝니다.</sub></p>

- **16종 프롬프트 테마(스킨과 독립)**: 알약·모카 / 파스텔 / 도쿄 나이트 / Gruvbox 레인보우 / Nord 폴라 / Dracula / Rosé Pine / Everforest / Kanagawa / 라테 라이트 / 플랫 컬러 / Jetpack 콕핏 / Pure 미니멀 / 초미니멀 한 줄 / 두 줄 / 플레인 텍스트.
- **겹쳐 적용 가능한 보정 5종(여러 개 동시 선택 가능)**: 언어 버전 숨기기 / 플레인 텍스트 기호·아이콘 제거 / 시간 끄기 / 명령 소요 시간 끄기 / 선행 빈 줄 제거.
- **스킨마다 독립된 터미널 색상**: 터미널 배경 / 커서 / 선택 영역 + 16 ANSI 전부 스킨에서 도출; 라이트 스킨의 컬러 글자도 대비 ≥ 3.5로 낮춰, 더 이상 밝은 바탕에 묻히지 않습니다.

<p align="center"><sub>▲ 내장 터미널: 탭은 프로젝트별 황금각 배색(레인보우), 상단 바의 공식 브랜드 아이콘 줄로 Claude / Codex / WeChat을 바로 실행할 수 있고, 프롬프트는 내장된 starship powerline입니다.</sub></p>

- **브랜드 아이콘 툴바**: Claude Code / Codex / WeChat 등의 실행 진입점을 **공식 브랜드 벡터 아이콘**으로 교체해 상단 바에 한 줄로 한눈에 보이게 했습니다; 나머지 동작 버튼(미리보기 팔로우 / 새 탭 / 전체 화면 / dock 전환 / 음소거…)도 모두 테마색을 따르는 단색 벡터 아이콘으로 다시 그렸습니다.
- **레인보우 탭**: 각 터미널 탭은 프로젝트별로 황금각으로 색을 정해, 여러 프로젝트가 나란히 있을 때 자동으로 어긋나 한 줄기 무지개를 이루어, 어느 터미널이 어느 프로젝트인지 한눈에 구분됩니다; 탭은 높이를 키우고 너비가 파일명에 맞춰 자동 조정되며, 탄력적으로 드래그해 위치를 바꿀 수 있습니다(인접 탭을 넘어서면 실시간으로 자리를 내줌).
- **「일반 터미널」 버튼**: Claude Code / Codex 옆에서 한 번의 클릭으로 현재 폴더에 깨끗한 셸(agent 없이)을 엽니다.
- **독립된 라운드 코너 터미널 카드**: 상단 내비게이션 바의 둥근 윗면, 배경이 현재 스킨의 바탕색을 따라 전체에 녹아듭니다(다크 스킨에서 더 이상 튀는 새까만 사각형이 아님), 라이트 스킨에서는 활성 탭을 옅게 물들이고 불필요한 하단 색 띠를 없앴습니다.

### 🗂 사이드바 · 진입점과 Agent 프로젝트 추가/삭제 가능

<p align="center">
  <img src="docs/screenshots/sidebar.png" alt="사이드바: 빠른 진입점 / 즐겨찾기 / Agent 프로젝트, 추가·삭제 및 드래그 정렬 가능 + 공식 사용량 패널" width="34%">
</p>
<p align="center"><sub>▲ 사이드바: 빠른 진입점 / 즐겨찾기 / Agent 프로젝트, 드래그로 추가·삭제·정렬 가능; 하단의 Agent 사용량 패널이 공식 한도를 상시 표시합니다.</sub></p>

- **빠른 진입점 추가/삭제 가능**: ➕로 현재 폴더를 추가하고, 마우스를 올려 ✕로 제거(기본 항목도 삭제 가능), 서버 측에 영속 저장.
- **Agent 프로젝트 추가/삭제 가능**: 수동으로 추가해 상단에 고정하고, 보고 싶지 않은 것은 숨길 수 있습니다(숨긴 뒤에는 스캔에서 다시 나타나지 않음); 목록에서 「즐겨찾기 / 빠른 진입점」으로 끌어다 놓거나, 목록 안에서 위아래로 드래그해 정렬을 사용자화할 수 있습니다(순서는 영속 저장).
- **사용량 패널 강화**: Claude Code 공식 한도(5시간 윈도우 / 주간 할당량)를 **항상 표시** —— 가져오면 진행 막대를 보여 주고, 못 가져오면 이유를 명시(미구독·미로그인 / 네트워크 제한 / 윈도우 데이터 없음) + 재시도; 한도에 근접하면(≥85%) **빨간 경고 막대 + 데스크톱 알림**(스로틀링으로 방해 최소화), 패널을 펼치면 자동으로 새로고침됩니다. 공식 한도 API에는 엄격한 속도 제한이 있어, 10분 캐시 + 제한 시 직전 데이터를 재사용하도록 바꿔, 더 이상 간헐적 속도 제한 때문에 「데이터 없음」이 표시되지 않습니다.

### 그 밖의 다듬기

- **빨간 ✕ 클릭 = 창 숨기기(macOS)**: 더 이상 창을 파괴해 터미널을 죽이지 않고, Dock을 클릭하면 그대로 다시 불러옵니다(터미널 / 상태 모두 유지), ⌘Q가 진짜 종료입니다.
- **창 상단 여백 전체를 드래그 가능**: 더 이상 브랜드 영역 한 귀퉁이만 잡을 수 있는 게 아닙니다; 왼쪽 위 신호등 버튼에 충분한 여백을 두어 logo와 겹치지 않습니다.
- **스크롤바 재제작**: 가늘고 둥글며 강조색을 따르고, 트랙 양 끝을 안쪽으로 들여 라운드 카드의 둥근 모서리를 피해 더 이상 튀어나오지 않습니다.
- **탭 커스텀 커서**: 터미널 탭 영역에 스킨 강조색을 따르는 작은 화살표 커서를 사용합니다.
- **자동 포트 양보**: 기본 4567이 사용 중이면 자동으로 다음 빈 포트 쌍으로 바꿔, 여러 인스턴스가 더 이상 충돌하지 않습니다.
- **커스텀 앱 아이콘 + 사이드바 logo**, 앱 이름을 Rurutia로 변경.

## 개인정보 보호와 보안

> 상류 FanBox와 동일하게, Rurutia는 그 보안 모델을 바꾸지 않습니다.

- 백엔드는 로컬 루프백 주소에서만 수신하고 Host 헤더를 검증해(DNS rebinding 차단), **데이터는 로컬을 벗어나지 않습니다**.
- 모든 프런트엔드 리소스(렌더러, 폰트, starship 바이너리 포함)가 로컬에 내장되어 **오프라인에서 완전히 사용 가능**합니다. 유일한 외부 네트워크 요청: Claude / Codex 사용량 API(선택)와 GitHub 업데이트 확인.
- HTML 미리보기는 origin이 격리된 샌드박스 iframe 안에서 렌더링되어 터미널 기능에 접근할 수 없습니다.
- 터미널 프롬프트는 ZDOTDIR 주입 방식으로, **당신의 어떤 dotfile도 쓰거나 고치지 않으며**, 제거 시 잔여물이 전혀 없습니다.
- 설정은 원자적 쓰기(temp + fsync + rename)로 데이터를 잃지 않으며; 삭제는 시스템 휴지통을 거칩니다(복구 가능).
