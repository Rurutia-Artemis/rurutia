**Coding Agent のコックピット —— [FanBox](https://github.com/alchaincyf/fanbox) をベースにした個人強化版**

## これは何か

[**FanBox**](https://github.com/alchaincyf/fanbox)（作者：[Huashu（花叔）](https://github.com/alchaincyf)）は、ローカルで動作する「**Coding Agent のコックピット**」です。ローカルファイルを閲覧／プレビュー／編集しながら、埋め込まれた本物のターミナルで Claude Code、Codex、あるいは任意の coding agent を走らせ、agent がどのファイルを書き換えたかをリアルタイムでハイライトします——**ファイルを見つける → agent を走らせる → 変更を見届ける**。これらすべてを 1 つのウィンドウで完結します。バックエンドは依存ゼロ、データはマシンの外に出ません。

> *「AI は午後の間に 10 個のプロジェクトを立ち上げてくれますが、そのあと二度と見つけられなくなります。FanBox はそれらを取り戻す手助けをします。」*

**Rurutia** は、私が FanBox v2.3.1（`c93a486`）をベースに行った**個人的なカスタマイズ**です。コア機能は 100% FanBox 由来で、私はビジュアル／フォント／配色を作り直し、スキンとターミナルプロンプトのシステム一式を追加し、日常の使い勝手を数十か所磨き込みました。以下では、まず**オリジナルプロジェクトの全機能**を説明し、そのうえで**私が具体的に何を変えたか**を説明します。

## 30 秒でわかる

| やりたいこと | Rurutia では |
|---|---|
| 午後に乱立した 10 個のプロジェクトを取り戻す | `⌘K` のグローバルあいまい検索 · フォルダに node/web/py/rs/go バッジが付き、種類が一目でわかる |
| agent に作業させつつ、何を変えたかも見える | 埋め込みの本物のターミナルで Claude Code / Codex を実行。書き込まれたファイルのカードがその場で光り、プレビューがリアルタイムで追従 |
| 昨日のセッションを再開する | プロジェクトを開いて履歴セッションを確認し、「▶ 再開」ワンクリックで `claude --resume` / `codex resume` がコンテキストを引き継ぐ |
| 公式の使用量を見張って超過を防ぐ | サイドバーに Claude / Codex の 5 時間ウィンドウ + 週次クォータを常時表示。上限に近づくと赤いバー + デスクトップ通知 |
| 気分に合わせて画面全体を着せ替える | 18 種類のカラースキン + 16 種類のターミナルプロンプトテーマで、UI / ターミナル / コードハイライトが一緒に変わる |

## オリジナル FanBox でできること（全機能）

> この部分は FanBox 本来の機能で、Rurutia はそのまま完全に保持しています。オリジナルの英語版説明は [`README.fanbox.md`](README.fanbox.md) を参照してください。

### 🗂 ファイル · 取り戻しとプレビュー
- **⌘K グローバルあいまい検索**：名前の一部を覚えていれば十分です。`⌘↵` でプロジェクトをエディタごと開き、`内容:キーワード` で全文検索に切り替えます。
- **鮮やかなソリッドアイコン**：どのファイルタイプも「それらしい見た目」に——PDF は赤、JS は黄、Markdown は青。写真と動画は実際の比率で表示します。
- **その場でプレビュー**：Markdown レンダリング、HTML のライブ表示、コードの構文ハイライト、画像／動画／PDF のインライン表示（HEIC を含む）、圧縮ファイルの中身一覧、透過画像には市松模様の背景を敷きます。
- **サムネイル高速化**：大きなフォルダでもスクロールとクリックが 0.1 秒以内に収まります。
- **プロジェクトバッジ**：フォルダカードに node / web / py / rs / go を表示し、午後に立ち上げた 10 個のプロジェクトでも種類が一目でわかります。

### 👀 agent が何を変えたかを見る
- **生きたダッシュボード**：agent がファイルを書くたびに、そのカードがその場で波紋を広げ、変更頻度に応じて発光・呼吸します。agent が書き進んだところに光が動いていきます。
- **フォローモード**：ワンクリックでファイルビュー + プレビューを agent が編集中のファイルに追従させます——コードは新しく書かれた行をハイライトして点滅し、HTML は書きながらリアルタイムでレンダリングし（ダブルバッファリングで白フラッシュなし）、Markdown もリアルタイムでレンダリングします。手動で操作した瞬間、コントロールはあなたに戻ります。
- **セッションリプレイ**：動画をスクラブするようにタイムラインをドラッグして、その間に agent が一歩ずつどのファイルを変えたかを再現します。
- **変更インボックス**：複数のプロジェクトをまたいで、このセッションで変更されたすべてのファイルを集約します。
- **Git 変更 diff**：Monaco の読み取り専用 DiffEditor が HEAD と現在の作業ツリーを並べて表示します。

### 🤖 Agent コックピット
- **プロジェクトメモリ**：どのプロジェクトフォルダを開いても、そこで AI が何をしたかがわかります——履歴セッション（あなたの最初の一文がタイトルになります）、各セッションで変更されたファイル、トリガーされた skill。「▶ 再開」ワンクリックで、埋め込みターミナルにて `claude --resume` / `codex resume` がコンテキストを引き継ぎます。
- **スクリーンショット直行便**：システムのスクリーンショットがディスクに保存された瞬間、直通カードが浮かび上がります——ターミナルの agent に渡す、プロジェクトの `素材/` に取り込む、あるいは先に注釈を付けてから送る、を選べます。
- **AI 整理**：AI はメタデータだけを見て整理の提案を出し（中身は読まず、ファイルシステムには触れません）、1 項目ずつ人が確認したうえで実行します + ロールバックログを書き出し、ワンクリックで全体を取り消せます。
- **リリースウィザード**：node プロジェクトなら、バージョン番号・CHANGELOG・パッケージング・プッシュ・GitHub Release をワンクリックで一気通貫します。
- **Skills の俯瞰**：マシン上のすべての agent skill を 1 つのビューで——トリガー統計、ヘルスチェック、context 予算、ファイルを消さずに有効／無効を切り替えるスイッチ。
- **Agent 使用量**：Claude Code 公式の 5 時間ウィンドウ／週次クォータ（`/usage` と同じソース）+ ローカルの token 統計。Codex の上限スナップショットも。
- **ディスク使用量の俯瞰**：`du` 基準の実使用量を棒グラフでランキング表示し、ドリルダウンできます。

### 🖥 ターミナル · agent を指揮する
- **本物の埋め込みターミナル**：node-pty + xterm.js（WebGL レンダリング）で、Claude Code / vim / htop を実行しても表示が乱れず、CJK の全角文字も正しく扱います。
- **ファイルをターミナルにドラッグ**：ファイル一覧からファイル／フォルダをターミナルにドラッグすると、パスを自動挿入して agent にコンテキストとして渡します。
- **クリックできるパス**：ターミナルに現れたファイルパスをそのまま開けます（スペースを含むスクリーンショット名、日本語名、折り返した長いパスもすべて認識します）。
- **選択してターミナルへ放り込む**：プレビューで一節のテキストを選び、「出典ファイル + フェンス」形式でワンクリックでターミナルに送ります。
- **状況把握**：タブのドットが agent の実行中／待機中／終了を表示します。あなたの番になるとターミナルの縁が呼吸して知らせ、長いタスクが完了するとシステム通知が飛びます。

### ✍️ 編集 · 見たままが結果
- **Markdown**：Milkdown Crepe が Notion ふうの WYSIWYG を提供し、入力を止めて 0.8 秒で自動保存します。
- **コード/JSON**：Monaco エディタ（VS Code と同じコア）。
- **画像注釈**：ブラシ／矢印／テキスト／墨消し、フォーマット変換、圧縮、解像度調整。
- **未保存ガード**：3 種類のエディタが、未保存のまま終了しようとするのを統一的にブロックします。

## Rurutia が改造したところ

> コア機能はすべて FanBox 由来です。以下は私が新規追加／作り直した部分です。全体は 4 つのことを軸にしています：**見た目がいい**、**見分けやすい**、**使いやすいターミナル**、**邪魔をしない**。

### 🎨 18 種類のカラースキン（マルチアクセント方式）

サイドバーの「スキン」をクリックするとスウォッチのグリッドが開いて切り替えられます。デフォルトは「ピクセルライト」。各スキンは「**ニュートラルなベース + 3 つの並立するアクセント色**（メイン：ボタン／アクティブ状態 · サブ：セクション見出しとリンク · 差し色：バッジ）+ 一組のセマンティックな状態色」で構成されており、色は鮮やかでありながら、役割分担によって調和を保ちます。本文／アクセント色／バッジの文字／ターミナルの 16 ANSI は**すべて WCAG のコントラスト検証を通過**しています。各スキンはメイン画面・サイドバー・ターミナル配色・コードハイライトを自動で適合させ、Monaco エディタの地色まで同じ色温度に合わせます。

着想は WeChat 公式アカウント「色所」の上質な配色コレクションから：ネオメンフィス / アシッドファッション / デジタルゼリー / 回路基板 / 虚空 / ラヴァオレンジ / 深海コア…… ライト 9 種・ダーク 9 種、合計 18 種類です。

</p>
<p align="center"><sub>▲ 18 種類のスキン一覧（ライト 9・ダーク 9）。一度切り替えるだけで、メイン画面 / サイドバー / ターミナル配色 / コードハイライトが一緒に変わります。</sub></p>

UI 全体も現代化しました：ヘアライン罫線、統一されたコーナー半径のリズム、フローティングのカプセル型セグメントコントロール、アクセント色のグロー、サイドバーの選択状態、控えめなトランジションアニメーション——すべて現在のスキンのアクセント色に追従します。インターフェース・ファイル名・コード・ターミナルは統一して **Maple Mono CN** を使用します（中国語 + 日本語の仮名を全字含み、woff2 を内蔵、オフラインでも利用可）。

### 🚀 ターミナルプロンプト（Starship 内蔵 · 16 種類のテーマ）

**すぐ使える powerline プロンプト**：署名・公証済みの starship + Nerd Font のアイコンフォントを内蔵しており、インストールしてターミナルを開けばそのまま powerline のピル（ディレクトリ / git の状態 / 言語バージョン / ♥ 時刻）が表示されます。**自分で starship を入れる必要も、`~/.zshrc` を設定する必要もありません**。

ZDOTDIR 注入で動作します：まずあなたの本物の dotfile を source し（PATH / エイリアスを寸分違わず再現、`claude` / `codex` もちゃんと見つかります）、そのうえに starship を重ねます——**この App のターミナルでのみ有効で、いかなる dotfile にも触れず、アンインストールしても残骸ゼロ**です。macOS + zsh のみ対応。

</p>
<p align="center"><sub>▲ サイドバーの「プロンプト」セレクター：テーマは 1 つ選択、修飾は複数選択して重ねがけ。切り替えは即時反映され、実行中のターミナルも Enter を押せば見た目が変わります。</sub></p>

- **16 種類のプロンプトテーマ（スキンとは独立）**：ピル・モカ / パステル / 東京ナイト / Gruvbox レインボー / Nord ポーラー / Dracula / Rosé Pine / Everforest / Kanagawa / ラテライト / フラット極彩 / Jetpack コックピット / Pure ミニマル / 極簡 1 行 / 2 行 / プレーンテキスト。
- **重ねがけできる 5 つの修飾（複数選択で並行）**：言語バージョンを隠す / プレーンテキスト記号・アイコンを外す / 時刻オフ / コマンド所要時間オフ / 先頭の空行を除く。
- **スキンごとに独立したターミナル配色**：ターミナルの背景 / カーソル / 選択範囲 + 16 ANSI をすべてスキンから導出します。ライトスキンの色文字もコントラスト ≥ 3.5 まで抑え、薄い地色に埋もれません。

### 🖥 ターミナル · ブランドアイコン + レインボータブ

<p align="center">
  <img src="docs/screenshots/terminal.png" alt="ターミナル：レインボーカラーのプロジェクトタブ + Claude/OpenAI/Codex/WeChat のブランドアイコンツールバー + powerline プロンプト" width="100%">
</p>
<p align="center"><sub>▲ 埋め込みターミナル：タブはプロジェクトごとに黄金角で配色（レインボー）、上部バーには公式ブランドアイコンが並び、Claude / Codex / WeChat を直接起動できます。プロンプトは内蔵の starship powerline です。</sub></p>

- **ブランドアイコンツールバー**：Claude Code / Codex / WeChat などの起動入口を**公式ブランドのベクターアイコン**に差し替え、上部バーに一列で並べて一目瞭然にしました。その他のアクションボタン（プレビュー追従 / 新規タブ / フルスクリーン / dock 切替 / ミュート…）もすべて、テーマ色に追従するモノクロのベクターアイコンに描き直しました。
- **レインボータブ**：各ターミナルタブはプロジェクトごとに黄金角で色を取り、複数プロジェクトを並べると自動でずれて一筋のレインボーになり、どのターミナルがどのプロジェクトか一目で見分けられます。タブは高さを増し、幅はファイル名に応じて自動調整され、弾力的にドラッグして並べ替えできます（隣のタブを越えた瞬間にリアルタイムで場所を譲ります）。
- **「普通のターミナル」ボタン**：Claude Code / Codex の隣に、現在のフォルダでクリーンな shell（agent なし）をワンクリックで開けます。
- **独立した角丸ターミナルカード**：ヘッダーのナビゲーションバーは上辺が円弧を描き、背景は現在のスキンの地色に追従して全体に溶け込みます（ダークスキンでも浮いた真っ黒の四角ではなくなりました）。ライトスキンではアクティブタブを淡い染め色に変え、余計な下部のカラーバーを取り除きました。

### 🗂 サイドバー · 導線と Agent プロジェクトを追加・削除可能

<p align="center">
  <img src="docs/screenshots/sidebar.png" alt="サイドバー：クイック入口 / お気に入り / Agent プロジェクト、追加・削除とドラッグ並べ替え対応 + 公式使用量パネル" width="34%">
</p>
<p align="center"><sub>▲ サイドバー：クイック入口 / お気に入り / Agent プロジェクトをドラッグで追加・削除・並べ替え可能。下部の Agent 使用量パネルは公式の上限を常時表示します。</sub></p>

- **クイック入口を追加・削除可能**：➕ で現在のフォルダを追加、ホバーして ✕ で削除（デフォルト項目も削除可）、サーバー側で永続化します。
- **Agent プロジェクトを追加・削除可能**：手動で追加して上部に固定、見たくないものを非表示（非表示にするとスキャンで再び出てきません）。一覧から「お気に入り / クイック入口」にドラッグしたり、一覧内で上下にドラッグして並び順をカスタマイズしたりもできます（順序は永続化）。
- **使用量パネルの強化**：Claude Code の公式上限（5 時間ウィンドウ / 週次クォータ）を**常時表示**——取得できればプログレスバー、取得できなければ理由を明示（未サブスク・未ログイン / ネットワーク制限 / ウィンドウデータなし）+ リトライ。上限に近づくと（≥85%）**赤い警告バー + デスクトップ通知**（スロットルして邪魔しません）、パネルを展開すると自動で更新します。公式の上限取得 API は厳しいレート制限があるため、10 分キャッシュ + レート制限時は前回データを流用する方式に変え、たまのスロットルで「データなし」と表示されることをなくしました。

### その他の磨き込み

- **赤い ✕ = ウィンドウを隠す（macOS）**：ウィンドウを破棄してターミナルを殺すのをやめ、Dock をクリックすればそのまま呼び戻せます（ターミナル / 状態はそのまま）。⌘Q が本当の終了です。
- **ウィンドウ上部の余白全体をドラッグ可能**：ブランド領域の一角しか掴めなかったのを改善。左上の信号機ボタンは十分な余白を確保し、logo と重なりません。
- **スクロールバーを作り直し**：細く丸く、アクセント色に追従。トラックの両端をインデントして角丸カードの角を避け、はみ出さなくなりました。
- **タブのカスタムカーソル**：ターミナルのタブ領域では、スキンのアクセント色に追従する小さな矢印カーソルを使います。
- **ポートの自動繰り下げ**：デフォルトの 4567 が使用中のとき、自動で次の空きポート対に切り替え、複数インスタンスでも衝突しなくなりました。
- **カスタムアプリアイコン + サイドバー logo**、アプリ名を Rurutia に変更。

### 🌐 インターフェース言語（7 言語）

- アプリ UI が **7 言語**に対応：简体中文 / 繁體中文 / English / 日本語 / 한국어 / Français / Español。サイドバーに「言語」セレクターを追加し、母語名のリストから選ぶだけで切り替えできます。ユーザーコンテンツ（プレビュー / エディター / ターミナル）は翻訳しません。

## プライバシーとセキュリティ

> 上流の FanBox と一貫しており、Rurutia はそのセキュリティモデルを変えません。

- バックエンドはローカルのループバックアドレスでのみリッスンし + Host ヘッダーを検証します（DNS rebinding を防ぐ）。**データはマシンの外に出ません**。
- すべてのフロントエンド資産（レンダラー、フォント、starship バイナリを含む）はローカルに内蔵されており、**オフラインでも完全に利用できます**。唯一の外向き通信：Claude / Codex の使用量 API（任意）と GitHub の更新チェック。
- HTML プレビューは origin を分離したサンドボックス iframe 内でレンダリングされ、ターミナルの機能には触れられません。
- ターミナルプロンプトは ZDOTDIR 注入で動作し、**あなたのいかなる dotfile も書き込まず・変更せず**、アンインストールしても残骸ゼロです。
- 設定はアトミック書き込み（temp + fsync + rename）で行うためデータを失わず、削除はシステムのゴミ箱を経由します（復元可能）。
