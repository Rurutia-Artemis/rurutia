# Rurutia

**Coding Agent のコックピット —— [FanBox](https://github.com/alchaincyf/fanbox) をベースにした個人強化版**

Claude Code / Codex にローカルで作業させながら、エージェントが触れたすべてのファイル、書き換えたすべての行を見届け、いつでも引き継げます。<br>

## これは何か

[**FanBox**](https://github.com/alchaincyf/fanbox)（作者：[Huashu (花叔)](https://github.com/alchaincyf)）は、ローカルで動作する「**Coding Agent のコックピット**」です。ローカルファイルを閲覧 / プレビュー / 編集しながら、組み込みの本物のターミナルで Claude Code、Codex、あるいは任意の coding agent を走らせ、エージェントが書き換えたファイルをリアルタイムにハイライトします——**ファイルを見つける → エージェントを走らせる → 変更を見届ける**、すべてが一つのウィンドウで完結します。バックエンドへの依存はゼロ、データはマシンの外に出ません。

> *「AI が午後いっぱいで 10 個のプロジェクトを立ち上げてくれる。そして、それらは二度と見つからなくなる。FanBox はそれらを取り戻す手助けをします。」*

**Rurutia** は、私が FanBox v2.3.1（`c93a486`）をベースに行った**個人的な改造**です。コア機能は 100% FanBox 由来で、私はビジュアル / フォント / 配色を作り直し、日常の使い勝手をいくつか磨きました。以下ではまず**オリジナルプロジェクトの全機能**を説明し、続いて**私が具体的に何を変えたか**を説明します。

## オリジナルの FanBox でできること（全機能）

> この部分は FanBox 本体の機能であり、Rurutia は完全に保持しています。オリジナルの英語版説明は [`README.fanbox.md`](README.fanbox.md) を参照してください。

### 🗂 ファイル · 再発見とプレビュー
- **⌘K グローバルあいまい検索**：名前の断片を覚えていれば十分です。`⌘↵` でエディタを使ってプロジェクトをまるごと開けます。`内容:キーワード` で全文検索に切り替えられます。
- **濃色のリアルなアイコン**：あらゆるファイルが「それ自身らしい見た目」に——PDF は赤、JS は黄、Markdown は青。写真や動画は実際の比率で表示されます。
- **その場でプレビュー**：Markdown レンダリング、HTML のリアルタイム成果物、コードの構文ハイライト、画像 / 動画 / PDF の組み込み表示（HEIC を含む）、アーカイブの中身リスト、透過画像はチェッカーボード背景で表示。
- **サムネイル高速化**：大きなフォルダのスクロールもクリックも 0.1 秒以内。
- **プロジェクトバッジ**：フォルダカードに node / web / py / rs / go を表示。午後いっぱいで立ち上げた 10 個のプロジェクトの種類が一目でわかります。

### 👀 エージェントが何を変えたかを見る
- **生きたダッシュボード**：エージェントがファイルを 1 つ書くたびに、そのカードがその場で波紋を広げ、変更頻度に応じて呼吸するように光ります。エージェントが書き進めるところまで光が走ります。
- **フォローモード**：ワンクリックでファイルビュー + プレビューが、エージェントが今編集中のファイルを追跡します——コードは新しく書かれた行がハイライトして点滅し、HTML は書きながらリアルタイムにレンダリング（ダブルバッファリングでホワイトフラッシュなし）、Markdown もリアルタイムにレンダリングされます。手動で操作すれば、すぐにコントロールがあなたに戻ります。
- **セッションリプレイ**：動画を見るようにタイムラインをドラッグして、この期間にエージェントが一歩ずつどのファイルを変更したかを再現します。
- **変更インボックス**：複数のプロジェクトをまたいで、本セッションで変更されたすべてのファイルを集約します。
- **Git 変更 diff**：Monaco の読み取り専用 DiffEditor で HEAD と現在のワークツリーを並べて表示します。

### 🤖 Agent コックピット
- **プロジェクトメモリ**：任意のプロジェクトフォルダを開くと、AI がそこで何をしたかを確認できます——過去のセッション（あなたの最初の一言がタイトルになります）、各セッションで変更したファイル、トリガーした skill。「▶ 続きから」をクリックすれば、組み込みターミナルで `claude --resume` / `codex resume` を実行し、コンテキストを引き継いで再開できます。
- **スクリーンショット直行便**：システムスクリーンショットが保存されると直行カードが浮かび上がります——ターミナル内のエージェントに渡す、プロジェクトの `素材/` に取り込む、あるいは注釈をつけてから送る、が選べます。
- **AI 整理**：AI はメタデータだけを見て整理の提案を出し（内容は読まず、ファイルシステムにも触れません）、一件ずつ人がチェックしてから実行 + ロールバックログを記録し、ワンクリックで一括取り消しできます。
- **リリースウィザード**：node プロジェクトはワンクリックでバージョン番号、CHANGELOG、パッケージング、プッシュ、GitHub Release をつなげて実行します。
- **Skills 透視**：このマシンのすべての agent skills を一つのビューで——トリガー統計、ヘルスチェック、context 予算、ファイルを削除しない有効 / 無効スイッチ。
- **Agent 使用量**：Claude Code 公式の 5h ウィンドウ / 週間クォータ（`/usage` と同じソース）+ ローカルの token 統計。Codex の上限スナップショット。
- **ディスク使用量透視**：`du` 基準の実際の使用量バーランキング、ドリルダウン可能。

### 🖥 ターミナル · エージェントを指揮する
- **本物の組み込みターミナル**：node-pty + xterm.js（WebGL レンダリング）で、Claude Code / vim / htop を走らせても画面が乱れず、中国語の全角文字も正しく表示されます。
- **ファイルをターミナルにドラッグ**：ファイルリストからファイル / フォルダをターミナルにドラッグすると、パスが自動挿入されエージェントにコンテキストとして渡されます。
- **パスをクリック可能に**：ターミナルに現れたファイルパスを直接クリックして開けます（スペースを含むスクリーンショット名、日本語名、折り返した長いパスも認識します）。
- **選択してそのままターミナルへ**：プレビューでテキストを選択し、ワンクリックで「ファイル出典 + フェンス」形式でターミナルに送れます。
- **状況認識**：タブの丸印でエージェントの実行中 / アイドル / 終了を表示。あなたの番になるとターミナルの縁が呼吸して知らせ、長いタスクが完了するとシステム通知を送ります。

### ✍️ 編集 · 見たままが結果
- **Markdown**：Milkdown Crepe が Notion 風の WYSIWYG を提供し、入力を止めて 0.8 秒で自動保存します。
- **コード / JSON**：Monaco エディタ（VS Code と同じカーネル）。
- **画像注釈**：ペン / 矢印 / テキスト / モザイク、フォーマット変換、圧縮、解像度調整。
- **未保存ガード**：3 種類のエディタで未保存のまま終了するのを統一して防ぎます。

## Rurutia で改造した点

> コア機能は完全に FanBox 由来です。以下は私が追加 / 作り直した部分です。

#### 🎨 ビジュアルと配色
- **18 種の「色所」高級感配色スキン**：ニューメンフィス / アシッドファッション / デジタルゼリー / 回路基板 / ヴォイド / ラヴァオレンジ / 深海コア…… サイドバーの「スキン」をクリックするとカラーカードのグリッドがポップアップして切り替えられます。デフォルトは「ピクセルライト」。各スキンはメイン画面、サイドバー、ターミナル、コードハイライトに自動で適応します。
- **UI 全体のモダン化**：ヘアライン枠線、統一された角丸のリズム、ホバーするカプセル型のセグメントコントロール、アクセントカラーのグロー、サイドバーの選択状態、控えめなトランジションアニメーション——すべて現在のスキンのアクセントカラーに追従します。

- **クイック入口の追加・削除**：➕ で現在のフォルダを追加、ホバーして ✕ で削除（デフォルト項目も削除可能）。サーバー側で永続化されます。
- **Agent プロジェクトの追加・削除**：手動で追加して最上部に固定、見たくないものは非表示に（非表示にするとスキャンで再び現れません）。

#### 📊 使用量パネルの強化
- Claude Code 公式の上限（5h ウィンドウ / 週間クォータ）を**常時表示**：取得できればプログレスバーを表示し、取得できなければ理由（未サブスクライブ・ログイン / ネットワーク制限 / ウィンドウデータなし）+ リトライを明記します。
- **上限に近づくと（≥85%）赤い警告バー + デスクトップ通知**（スロットリングして邪魔しません）。パネルを展開すると自動で更新されます。

#### 🖥 ターミナル & アイコン
- ターミナル上部バーの 10 個のアクションボタンを**すべて単色ベクターアイコンに描き直し**（テーマカラーに追従）。タブを高くし、幅をファイル名に合わせて自動調整します。
- カスタムアプリアイコン + サイドバーロゴ、アプリ名を Rurutia に変更しました。

#### ⚙️ その他
- **ポートの自動繰り上げ**：デフォルトの 4567 が使用中の場合、自動で次の空きポートのペアに切り替え、複数インスタンスでも競合しなくなります。

#### 🪄 インタラクションの細部の磨き込み
- **ウィンドウ上部の余白全体をドラッグ可能に**：ブランド領域の一角しか掴めない、ということがなくなりました。左上の信号機ボタンは十分な余白を確保し、ロゴと重なりません。
- **ターミナルを独立した角丸カードに**：ヘッダーナビゲーションバーの上端を丸くし、背景は現在のスキンのベースカラーに追従して全体に溶け込みます（ダークスキンでも、唐突な真っ黒の四角ではなくなりました）。
- **ターミナルタブの弾力あるドラッグ並べ替え**：横にドラッグし続けると、隣のタブを越えた瞬間にリアルタイムで場所を譲るので、目標の位置にぴったり合わせる必要がありません。
- **サイドバーのドラッグ**：Agent プロジェクトを直接「お気に入り / クイック入口」にドラッグできるほか、リスト内で上下にドラッグして並び順をカスタマイズできます（順序は永続化）。更新時刻は統一して右揃えに。
- **スクロールバーの作り直し**：細い丸型で、アクセントカラーに追従。トラックの両端は角丸カードの角を避けてインデントされ、はみ出さなくなりました。
- **グローバルに Maple Mono を強制**：一部のハードコードされたフォントの隅々まで押さえ、中 / 英 / 日の文字を本当に完全統一しました。
- **Claude 使用量の安定化**：公式の上限取得 API には厳しいレート制限があるため、10 分キャッシュ + レート制限時は前回データを流用するように変更し、たまのレート制限で「データなし」と表示されることがなくなりました。

## インストール

**macOS（Apple Silicon / arm64）**

1. [**Releases**](../../releases) から最新の `Rurutia-*.dmg` をダウンロードします。
2. dmg を開き、**Rurutia** を「アプリケーション」にドラッグします。
3. ダブルクリックで開けば使用できます。

> ✅ 本バージョンは **Apple Developer ID 証明書による署名 + Apple 公証（notarization）+ hardened runtime** 済みです——Releases からダウンロードして**ダブルクリックでそのままインストール・使用**でき、「開発元を確認できません」の警告は出ず、追加の操作は一切不要です。

## ソースからのビルド

```bash
npm install
npm run rebuild        # node-pty を Electron の ABI に再コンパイル

# 未署名のローカルビルド（最も簡単、自分用）：
CSC_IDENTITY_AUTO_DISCOVERY=false npx electron-builder --mac --dir -c.mac.identity=null
# 成果物：dist/mac-arm64/Rurutia.app
```

## 変更の構成（パッチ方式）

上流の FanBox の継続的なアップデートに追従できるよう、変更はできるだけ**追加方式**にして、新バージョンが出るたびに `git rebase` で再適用しやすくしています：

- **新規ファイル**（衝突しない）：`public/ui-patch.css`（すべての UI / フォント）、`public/themes-patch.js`（18 種のスキン）、`public/vendor/fonts/maple/*`、`public/vendor/icons/`、`public/logo.png`、`public/favicon.png`。
- **編集した上流ファイル**（少数、要注意）：`public/index.html`、`public/app.js`、`server.js`、`electron/main.js`、`package.json` + `build/icon*`。

完全な変更リスト + 「上流に新バージョンが出た後の再適用手順」は [`RURUTIA-PATCH.md`](RURUTIA-PATCH.md) を参照してください。

## プライバシーとセキュリティ

> 上流の FanBox と同様、Rurutia はそのセキュリティモデルを変更していません。

- バックエンドはマシンのループバックアドレスのみでリッスンし、Host ヘッダーを検証します（DNS rebinding を防止）。**データはマシンの外に出ません**。
- すべてのフロントエンドリソース（レンダラーやフォントを含む）はローカルに内蔵されており、**オフラインで完全に利用可能**です。唯一の外部通信は、Claude 使用量 API（任意）と GitHub のアップデートチェックのみです。
- HTML プレビューは origin を分離したサンドボックス iframe 内でレンダリングされ、ターミナルの機能には触れられません。
- 設定はアトミック書き込み（temp + fsync + rename）で行われ、データが失われません。削除はシステムのゴミ箱経由（復元可能）です。

## 技術アーキテクチャ

| レイヤー | 使用技術 |
|---|---|
| バックエンド | 依存ゼロの Node.js `server.js`（ファイル API + 静的配信 + サムネイル） |
| デスクトップシェル | Electron 33 + node-pty（asarUnpack でネイティブモジュール） |
| ターミナル | xterm.js + WebGL + unicode11 |

| フォント | Maple Mono CN（woff2 内蔵） |
| パッケージング | electron-builder → 署名 + 公証済みの arm64 `.dmg` |
